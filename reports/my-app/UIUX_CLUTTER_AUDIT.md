# UI/UX 난잡도 점검 보고서

작성일: 2026-05-26

## 이번 즉시 수정

- 대상: `/tablet/products`, `/products`가 공유하는 `components/storefront/TabletMallPages.tsx`
- 조치:
  - 기존 상단 알약 배지 위치에 `조리원명 + 객실`, `QR 코드 + 만료 시각`, `장바구니 연결` 정보를 배치했다.
  - 하단의 별도 3개 카드 `조리원`, `QR 세션`, `결제 상태/실결제 없음` 영역은 제거했다.
  - `Firebase 상품` 데이터 소스 배지는 유지했다. 이는 실제 Firestore read 여부 확인에 필요한 운영 진단 표시다.

## 난잡도 원인 패턴

1. 같은 의미의 상태 표시가 여러 줄에 반복된다.
   - 예: 상단 배지, 하단 정보 카드, 개발자 진단 패널이 동시에 노출됨.

2. 사용자용 정보와 개발자용 정보가 같은 시각 레벨로 표시된다.
   - 예: 고객 쇼핑 화면에 `Firebase`, `env`, `PG`, `mock`, `source` 같은 내부 상태가 너무 크게 보임.

3. 카드가 많아 첫 화면에서 상품/행동 버튼보다 운영 상태가 먼저 보인다.
   - 쇼핑몰 화면은 상품, 가격, 혜택, 장바구니/QR CTA가 우선이어야 한다.

4. 관리자/기업/조리원 화면은 정보 밀도는 필요하지만, 주의/차단/모의 배지가 모든 카드에 반복된다.
   - 반복 경고는 상단 1회와 우측 위험 패널로 묶는 편이 낫다.

## 우선 개선 대상 페이지

### 1. 고객/태블릿 쇼핑

| route | 문제 | 개선 방향 |
| --- | --- | --- |
| `/tablet/products` | 상단 운영 상태와 상품 홈 정보가 섞임 | 완료: 조리원/QR 정보를 상단 배지로 통합, 중복 카드 제거 |
| `/products` | `/tablet/products`와 같은 UI를 공유하므로 동일 영향 | 완료: 공유 컴포넌트 수정으로 반영 |
| `/tablet/products/[id]` | 개발자 진단/상품 상세/구매 CTA가 같은 위계로 보일 수 있음 | 상품 이미지/가격/CTA 우선, 개발자 정보는 접힘 또는 하단으로 이동 |
| `/tablet/cart` | 장바구니와 QR 생성 안내가 기능 카드처럼 분산됨 | 수량/옵션/수령방식/합계/QR 생성 CTA 순서로 단순화 |
| `/tablet/qr` | QR 정보, 상품 요약, 고객 이동 링크가 중복될 수 있음 | QR 코드와 만료/주문요약만 남기고 진단 배지는 축소 |

### 2. 고객 모바일 QR

| route | 문제 | 개선 방향 |
| --- | --- | --- |
| `/q/[code]` | 결제 가능/QR 상태/데이터 소스/상품 요약이 카드별로 쌓임 | 고객에게는 상태와 결제 CTA만 우선 노출, 데이터 소스는 작은 개발자 패널로 이동 |

| `/orders/guest` | mock 입력 UI와 개인정보 경고가 분리됨 | 주문번호/휴대폰 입력과 개인정보 안내를 하나의 폼 카드로 통합 |

## 디자인 정리 원칙

1. 고객 쇼핑 화면에서는 내부 시스템 용어를 첫 화면에서 숨긴다.
2. 조리원/객실/QR 만료처럼 고객 행동에 필요한 정보는 상단 한 줄 배지로 통합한다.
3. 결제 상태가 `실결제 없음`인 것은 고객 상품 홈에 크게 노출하지 않는다. 결제/checkout 화면에서만 안내한다.
4. Firebase/PG/mock/fallback 진단은 개발자 패널 또는 하단 소형 영역으로 이동한다.
5. 관리자 화면은 반복 경고를 줄이고 `요약 -> 목록 -> 상세/수정` 흐름을 고정한다.
6. 기업 상품 등록은 긴 단일 페이지가 아니라 단계형 등록 흐름으로 바꾼다.
7. 조리원 화면은 실무자가 오늘 처리할 객실/QR/수령 업무만 첫 화면에 둔다.

## 다음 수정 우선순위

1. `/tablet/products/[id]` 상세 화면: 구매 CTA와 개발자 진단 분리.
2. `/q/[code]/checkout`: 서버 검증/PG 준비/고객 입력 흐름을 단계형으로 재정렬.
3. `/company/products/new`: 법적 고지와 KC 인증을 stepper로 분리.
4. `/admin/marketing/banners`: CMS 등록/목록/미리보기 탭 분리.
5. `/nursery/dashboard`: 오늘 처리 업무 중심 대시보드로 축약.
