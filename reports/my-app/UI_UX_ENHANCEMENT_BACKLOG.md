# my-app UI/UX 고도화 백로그

## 목적

현재 mock/test 베타 화면을 운영 연결 없이 더 전문적인 폐쇄몰/관리자 제품처럼 보이게 만들기 위한 공통 기준이다. 이 문서는 실제 코드 구현이 아니라 병렬 트랙에서 참고할 고도화 큐다.

## 공통 UI 상태

### 빈 상태 UI

- 데이터가 없을 때 단순 문장만 표시하지 않고 제목, 짧은 설명, 다음 행동 버튼을 제공한다.
- 예시 상태:
  - 등록된 상품 없음
  - 승인 대기 입점사 없음
  - 조회 조건에 맞는 주문 없음
  - 활성 QR 세션 없음
  - 정산 대기 내역 없음

### 오류 상태 UI

- 오류 원인과 사용자가 다음에 할 수 있는 행동을 분리해서 보여준다.
- mock/test 베타에서는 실제 장애가 아니라 시나리오 확인용 상태로 표시한다.
- 예시 상태:
  - QR 세션 만료
  - mock 결제 실패
  - 주문번호와 휴대폰번호 불일치
  - 권한 부족 mock
  - 외부 API 미연동

### 위험 상태 배지

- 위험 상태는 색상만으로 구분하지 않고 텍스트 라벨을 함께 둔다.
- 권장 라벨:
  - `검토 필요`
  - `만료`
  - `결제 실패`
  - `정산 보류`
  - `재고 부족`
  - `연동 대기`
  - `권한 확인`

## 반응형 기준

### 모바일

- 고객 QR 화면은 단일 컬럼으로 구성한다.
- 결제 진입 CTA는 화면 하단에 고정하거나 마지막 요약 영역에 둔다.
- 긴 표 대신 카드형 요약을 사용한다.

### 태블릿

- 상품 목록은 2~3열 카드 중심으로 구성한다.
- 객실/조리원 배너와 장바구니 CTA가 항상 쉽게 보이게 한다.
- 터치 대상은 작게 만들지 않는다.

### 데스크톱 관리자

- 표, 필터, KPI, 상세 패널의 정보 밀도를 높인다.
- 반복 카드 남발보다 스캔 가능한 표와 상태 배지를 우선한다.
