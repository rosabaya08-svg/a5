# 인피니 기업별 MID / 상품등록 권한 정리 바이브코딩 프롬프트

목표:
- 최고관리자는 기업 회원가입 승인과 인피니 MID 입력만 담당한다.
- 기업 어드민은 상품등록, 상품 고시, 카테고리 분류, 가격 정책, 배송/반품/AS, 증빙 업로드, 승인 요청을 담당한다.
- 기업 어드민은 인피니 MID와 결제 모듈 키를 입력할 수 없고 조회만 한다.
- 태블릿/고객 결제는 반드시 상품의 companyId에 연결된 기업별 MID와 결제 모듈 키를 사용한다.

최고관리자 역할:
- 회원가입 승인대기 목록에서 기업 가입 정보를 확인한다.
- 기업 사업자 정보, 대표자, 담당자, CS, 반품지, 제출서류 정보를 인피니에 전달한다.
- 인피니가 승인한 뒤 기업별로 발급한 MID와 결제 모듈 키를 해당 기업 행에 입력한다.
- 상태값은 `not_applied`, `in_review`, `mid_issued`, `active`, `blocked` 중 하나로 관리한다.
- 별도 전역 PG 설정판에서 대표 MID 하나를 모든 기업에 넣는 구조를 만들지 않는다.
- 위드씨 또는 플랫폼 사업자 MID는 입점 기업 상품 결제 fallback으로 사용하지 않는다.

기업 어드민 역할:
- 상품등록 화면에서 카테고리 분류, 상품 기본정보, 상세페이지, 옵션/SKU, 가격정책, 재고, 상품정보제공고시, KC/인증, 배송/반품/AS, 판매자 고지를 입력한다.
- 상품은 기업 어드민에서 draft 저장 또는 승인 요청으로 제출한다.
- 최고관리자는 제출된 draft를 승인/반려만 한다.
- 기업 어드민의 PG 화면은 인피니 PG 상태, 마스킹된 MID, 운영 가능 여부만 보여준다.

PG 결제 정책:
- 한 결제 QR에는 한 기업, 한 MID 상품만 포함한다.
- 장바구니는 기업별 결제 묶음 카드로 보여준다.
- 각 기업 카드에는 기업명, MID 마스킹, 결제 예정 금액, 품목 수, QR 생성 버튼을 둔다.
- 고객 안내 문구는 반드시 노출한다: "업체별 공동구매로 기업별로 구매가 가능합니다."
- 고객이 특정 기업 QR 결제를 완료하면 해당 기업 상품만 장바구니에서 제거한다.
- 남은 기업 상품이 있으면 장바구니에서 다음 업체 결제 QR을 이어서 생성할 수 있게 안내한다.
- 기업별 MID 또는 결제 모듈 키가 없거나 active 상태가 아니면 해당 기업 결제 QR 생성 버튼을 비활성화한다.
- 전체 장바구니 총액 결제 버튼은 제공하지 않는다.

데이터 기준:
- 기업별 결제 정보 저장 위치는 `companies/{companyId}`이다.
- 필수 필드 후보:
  - `pg_provider = "infiny"`
  - `pg_merchant_id`
  - `pg_module_key`
  - `pg_merchant_status`
  - `infiny_mid`
  - `infiny_mid_status`
- 결제 ready 단계는 QR item의 단일 companyId를 기준으로 `companies/{companyId}`에서 MID와 모듈 키를 조회한다.
- 결제 confirm 단계는 ready 단계에서 저장된 companyId, MID, 모듈 키가 실제 장바구니 companyId와 일치하는지 검증한다.
- real PG 어댑터가 붙기 전에는 실제 승인 처리로 주문 완료를 만들지 않는다.

UI 정리:
- `/admin/companies`: 회원가입 승인대기, 인피니 전달 상태, 기업별 MID/결제 모듈 키 입력
- `/admin/products`: 기업이 제출한 상품 draft 검수, 승인/반려
- `/admin/payments`: 결제 상태와 장애 모니터링
- `/company/products/new`: 상품등록, 고시, 카테고리, 가격, 옵션, 증빙 입력
- `/company/products/preview`: 기업 상품 draft 미리보기
- `/company/onboarding` 또는 기업 대시보드: 인피니 PG 상태 읽기 전용 확인

금지:
- 최고관리자 전역 PG 설정 화면에 상품등록/고시/카테고리 입력 기능을 넣지 않는다.
- 기업 어드민에 MID 입력 필드를 넣지 않는다.
- 위드씨 대표 MID를 입점 기업 결제 fallback으로 사용하지 않는다.
- 공식 인피니 승인 API가 없는데 mock 승인으로 paid/order 완료를 만들지 않는다.
