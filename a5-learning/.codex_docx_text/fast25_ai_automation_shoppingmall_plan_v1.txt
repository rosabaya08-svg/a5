1. 문서 목적과 판단 기준
이 문서는 업로드된 Codex/Next.js/자동화/쇼핑몰 설계 문서를 기준으로, 기업입점형 QR 결제 쇼핑몰을 AI 자동화로 최단 기간 생성하기 위한 실행 로직을 정리한 문서입니다.
확인됨
업로드된 자동화 문서는 자동화를 운영 시스템 무인완성이 아니라 개발 브랜치 또는 worktree 안에서 안전 기능을 생성하고, 테스트하고, 오류를 검토하고, 보고서를 남기는 구조로 정의합니다.
확인 불가
AI 자동화만으로 오류 0건, 운영 무사고, 실결제/환불/정산 지급/운영 배포까지 완성된 상용 오픈을 보장하는 것은 확인 불가입니다.
따라서 이 문서의 목표는 상용 오픈이 아니라 “테스트 가능한 베타”입니다. 운영 PG, 실결제, 실환불, 정산 지급, 운영 배포, 법무/약관 최종 판단은 범위에서 제외합니다.

기존 기준 = 47작업일 FAST-25 = 25작업일 단축 작업일 = 47 - 25 = 22작업일 단축률 = 22 / 47 × 100 = 46.8% 데모 MVP = 15작업일 단축 작업일 = 47 - 15 = 32작업일 단축률 = 32 / 47 × 100 = 68.1%
Sanity Check
15작업일은 데모 MVP입니다. 기업입점 쇼핑몰 베타라고 부르려면 최소 25작업일이 현실적이며, 오류 리스크를 낮추려면 32작업일 버전이 더 안전합니다.
11. Codex 실행 프롬프트
11.1 FAST-25 마스터 프롬프트
너는 산후조리원 폐쇄몰 기반 기업입점형 QR 결제 쇼핑몰의 FAST-25 개발 에이전트다. 목표: 25작업일 안에 테스트 가능한 베타를 생성한다. 절대 제외: - PG 운영 MID/Secret 등록 - 실결제 - 운영 환불/부분취소 - 정산 지급 - 운영 배포 - Secret Key 변경 - 운영 DB 삭제/마이그레이션 - 개인정보/약관 최종 판단 우선순위: 1. AGENTS.md / PROJECT_RULES / DB_SCHEMA / STATUS_MODEL / AUTO_MODE_POLICY 2. Next.js App Router 기반 관리자/기업 Admin/태블릿/QR 화면 3. mock API와 adapter interface 4. QR 세션, 주문, 결제, 정산은 mock/test 초안까지만 5. lint/build/test/browser smoke test 6. AUTO_REPORT / BLOCKERS / NEXT_TASKS 작성 진행 규칙: - 기능 1개 단위로 작업한다. - 수정 전 계획을 먼저 출력한다. - 요청 범위 외 수정하지 않는다. - A등급은 생성/테스트/최대 2회 수정한다. - B등급은 초안/테스트/위험 보고까지만 한다. - C등급은 코딩하지 않고 BLOCKERS.md에 기록한다. - 공식 문서 없는 외부 API는 만들지 말고 mock adapter만 만든다. - 실제 테스트하지 않은 항목을 성공으로 보고하지 않는다. 먼저 오늘 작업 후보 5개를 A/B/C 등급, 예상 소요시간, 위험도, 병렬 가능 여부로 정리하고, 1순위 작업부터 시작하라. 아직 코드는 수정하지 마라.
11.2 방향 이탈 방지 프롬프트
아래 범위 밖의 작업은 코드로 구현하지 마라. 요청 범위: [오늘 작업 범위] 수정 가능: [허용 파일/폴더] 수정 금지: [금지 파일/기능] 범위 밖에서 더 좋은 개선안이 보이면 NEXT_TASKS.md에 제안만 남겨라. 결제/환불/정산/보안/운영 배포/Secret Key/운영 DB 관련 작업이 필요하면 즉시 중단하고 BLOCKERS.md에 기록하라.
11.3 거짓 코딩 방지 프롬프트
공식 문서 또는 프로젝트 문서로 확인되지 않은 API, endpoint, parameter, response format, Secret Key 이름을 만들지 마라. 확인되지 않은 연동은 mock adapter로만 작성하라. 실제 연동이 필요하면 필요한 공식 문서, 테스트키, 계약 정보, 승인 절차를 BLOCKERS.md에 작성하라. 테스트를 실행하지 않았다면 성공했다고 쓰지 마라. 실패한 테스트는 실패했다고 기록하라.
