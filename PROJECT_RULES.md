# Project Rules

작성일: 2026-05-19

## 1. 프로젝트 정의

이 프로젝트는 산후조리원 객실 태블릿에서만 상품을 탐색하고 장바구니를 구성한 뒤, 고객 또는 보호자가 QR/조르기 링크로 비회원 mock 결제 흐름에 진입하는 개발용 폐쇄몰 베타다. 입점 기업은 상품, 재고, 주문, 배송, 매출, 입금 현황을 mock 데이터로 확인하고, 최고관리자는 입점사, 조리원, 객실, 태블릿, 상품 승인, 주문, QR, 결제, 정산, 알림, 감사 로그를 통합 확인한다.

## 2. 현재 범위

현재 작업 범위는 운영 완성이 아니라 mock/test 베타 생성이다.

허용:

- Next.js App Router 화면
- TypeScript 타입
- mock 데이터
- mock API 함수
- mock/test adapter
- 개발용 UI 컴포넌트
- 정책/상태/보안 문서
- lint/build 검증

금지:

- 실제 Firebase 연결
- `.env`, `.env.*`, Secret Key, service account, private key 생성
- 실제 PG 연동
- 실결제, 실환불, 운영 부분취소
- 정산 지급 완료 처리
- 운영 배포
- 외부 API 운영 호출

## 3. 사용자/권한 구조

| 주체 | 계정/세션 | 역할 |
| --- | --- | --- |
| 최고관리자 | 관리자 계정 | 전체 운영, 승인, 정산 검산, 감사 |
| 기업 Admin | 입점사 계정 | 상품, 재고, 주문, 배송, 매출, 입금 확인 |
| 산후조리원 Admin | 조리원 계정 | 객실, 태블릿, 현장수령, QR 이력 확인 |
| 태블릿 | device/session | 폐쇄몰 탐색, 장바구니, QR/조르기 생성 |
| 고객/보호자 | 비회원 | QR 결제 흐름, 주문조회 |

## 4. 구현 원칙

- 문서 기반으로만 구현한다.
- 외부 연동은 interface와 mock adapter까지만 만든다.
- 모든 금액 표시는 mock 데이터 기준이며 운영 결제 근거가 아니다.
- 결제 금액은 실제 구현 시 프론트 값을 신뢰하지 않고 서버 snapshot 기준으로 재계산해야 한다.
- 정산은 `orders` 총액이 아니라 `order_items` 기준으로 계산해야 한다.
- QR 세션은 2~3시간 만료, 1회성 사용, 재사용 차단을 전제로 모델링한다.
- 권한은 role, company_id, nursery_id, room_id, tablet_id 기준으로 격리한다.

## 5. 현재 베타의 한계

이번 베타는 실제 인증, 실제 DB, 실제 결제, 실제 알림, 실제 배송조회, 실제 외부 재고 API를 사용하지 않는다. UI와 mock 데이터, mock adapter는 운영 전환 설계 검토를 위한 개발 산출물이다.
