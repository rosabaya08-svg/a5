최종 기준
Codex는 “코드를 대신 작성하는 도구”가 아니라 “정해진 작업 범위 안에서 파일을 읽고, 수정하고, 테스트하고, 보고서를 남기는 코딩 실행자”로 사용합니다. Next.js는 “공식 문서와 프로젝트 규칙을 먼저 읽고 구현하는 대상 프레임워크”로 정리합니다.
Codex 코딩 핵심만 정리
2.1 Codex가 필요한 이유
대화식 요청만으로도 코드 생성은 가능하지만, 실제 프로젝트를 안전하게 수정하려면 작업 폴더, 수정 권한, 승인 정책, 검증 방식이 필요합니다. Codex 명령어와 slash command는 코딩 문법이 아니라 “작업 환경 제어 장치”입니다.

2.4 AGENTS.md의 역할
AGENTS.md는 Codex가 작업 전에 읽는 프로젝트 작업 규칙입니다. 대표님 프로젝트에서는 “추측 코딩 금지, 공식 문서 우선, 기능 1개씩, 결제/정산/환불/배포 금지, 수정 후 보고서 작성”을 여기에 넣어야 합니다.
# AGENTS.md 핵심 템플릿 ## 작업 원칙 - Next.js 작업 전 node_modules/next/dist/docs/의 관련 문서를 먼저 확인한다. - 요청 범위 외 수정 금지. - 기능 1개 단위로 계획 -> 수정 -> 검증 -> 보고 순서로 진행한다. - 기존 기능 삭제, 임시 우회, 추측 외부 API 구현 금지. ## 금지 영역 - 운영 배포 금지. - 실결제, 운영 환불, 부분취소, 정산 지급 금지. - Secret Key 프론트 노출 금지. - 운영 DB 삭제/마이그레이션 자동 실행 금지. ## 완료 보고 - 수정 파일, 추가 파일, 실행 테스트, 실패 항목, 사람 확인 필요 항목, 다음 작업을 요약한다.
3. Codex 작업 흐름과 필수 산출물
3.1 표준 작업 흐름
1. AGENTS.md, PROJECT_RULES.md, DB_SCHEMA.md, STATUS_MODEL.md, SECURITY_POLICY.md를 먼저 읽게 합니다.
2. 오늘 작업을 기능 1개로 제한합니다.
3. 수정 전 /plan으로 수정 파일, 영향 범위, 테스트 계획을 받습니다.
4. 승인 후 관련 파일만 최소 수정합니다.
5. lint, build, unit test, UI smoke test를 실행합니다.
6. 안전 영역의 오류만 최대 2회 수정합니다.
7. 위험 영역이 필요하면 중단하고 BLOCKERS.md에 기록합니다.
8. AUTO_REPORT.md와 NEXT_TASKS.md를 남깁니다.

Next.js 핵심만 정리
Next.js는 대표님 프로젝트의 관리자 웹, 기업 Admin, 산후조리원 Admin, 고객 QR 결제 웹을 구현하는 기준 프레임워크로 정리합니다. 새 화면은 App Router 기준으로 설계하고, Codex가 오래된 지식이 아니라 설치된 Next.js 버전의 문서를 읽고 구현하게 하는 것이 핵심입니다.

9.1 마스터 프로젝트 이해
너는 산후조리원 폐쇄몰 기반 기업 입점형 QR 결제 쇼핑몰 시스템의 메인 개발 에이전트다. 이 프로젝트는 결제, 기업 Admin, 입점형 정산, 폐쇄몰, 산후조리원 객실/태블릿 출처 추적, QR 결제, 비회원 주문조회, 알림, 외부 재고/API 연동이 포함된 상용 커머스 시스템이다. 절대 원칙: - 추측 코딩 금지 - 운영 배포 금지 - 실결제/환불/정산 지급 금지 - Secret Key 프론트 노출 금지 - 결제/환불/정산 서버 처리 - company_id/nursery_id/tablet_id 격리 완료 시 수정 파일, 추가 파일, 테스트 결과, 실패 항목, 사람 확인 필요 항목, 다음 작업 명령어를 보고한다.
