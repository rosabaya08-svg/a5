본문은 Codex 코딩 실행, Next.js 구현, 쇼핑몰 도메인 제약, 안전 자동화, 하네스 연속성에 필요한 내용만 남깁니다.
1.2 확인 불가 또는 추가 확인 필요
PG 운영 MID/Secret, 실제 환불/부분취소, 정산 지급, 운영 배포, 카카오 알림톡 템플릿 승인, 외부 재고 API 운영 규격은 업로드 파일 기준으로도 사람 승인과 공식 계약/키/문서 확보가 필요합니다.
이 문서는 코딩 실행 지시서이며, 법무·세무·개인정보·운영 배포 최종 판단 문서가 아닙니다.
영상 대본 원문 전체를 근거로 삼지 않고, 업로드 파일과 대화에서 정리된 Codex/Next.js/하네스 관련 내용만 문서화했습니다.
최종 기준 Codex는 “운영 시스템을 무인으로 완성하는 도구”가 아니라, 개발 브랜치 또는 worktree 안에서 정해진 범위의 파일을 읽고, 수정하고, 테스트하고, 보고서를 남기는 코딩 실행자로 사용합니다.

4. Codex 하네스 구조
4.1 하네스 정의
하네스는 Codex가 덜 추측하고, 덜 빗나가고, 계속 이어서 작업하고, 테스트와 리뷰까지 수행하도록 만드는 외부 작업 환경입니다. 대표님 프로젝트에서는 “AGENTS.md + 프로젝트 규칙 문서 + 자동화 등급 + 산출물 보고서 + worktree + 검증 루프”가 하네스의 핵심입니다.

6. Next.js 구현 기준
6.1 핵심 원칙
새 화면은 App Router 기준으로 설계합니다.
Codex는 Next.js 작업 전 설치된 프로젝트의 bundled docs, 즉 node_modules/next/dist/docs/의 관련 문서를 먼저 확인해야 합니다.
서버 데이터 조회는 Server Component를 우선 사용하고, 상태·클릭·입력·브라우저 API가 필요한 부분만 Client Component로 분리합니다.
상품 등록, 승인 요청, 송장 입력, QR 생성 요청 같은 데이터 변경은 Server Action을 우선 검토합니다.
PG callback, 알림톡 webhook, 배송조회 adapter, 외부 재고 sync endpoint 같은 외부 호출은 Route Handler 또는 서버 adapter로 처리합니다.
저장/수정 후 필요한 경로만 revalidate하고, 대시보드는 전체 원장을 직접 합산하지 않고 summary 데이터를 사용합니다.

7. 쇼핑몰 구현 핵심 반영
7.1 시스템 한 줄 정의
산후조리원 객실 태블릿에서만 상품을 장바구니에 담고, 고객은 QR 또는 조르기 링크로 비회원 간편결제를 진행하며, 입점 기업은 상품·재고·주문·배송·매출·입금 현황을 관리하고, 최고관리자는 전체 운영·결제·정산·알림·보안을 통제하는 폐쇄몰형 입점 커머스 시스템입니다.

9. 코딩 연속성 기능 분석
9.1 문서 기반 연속성
AGENTS.md는 시작 지침이고, docs/는 프로젝트 지식 원장입니다.
AUTO_REPORT.md는 오늘 한 일, BLOCKERS.md는 중단 사유, NEXT_TASKS.md는 다음 작업 후보를 기록합니다.
큰 기능은 docs/exec-plans/active/에 계획 파일을 두고, 완료되면 docs/exec-plans/completed/로 이동합니다.
기억 기능이나 대화 기록보다 저장소 안에 남는 문서가 더 안전한 연속성 장치입니다.

상품 UI, 주문 UI, QR 초안, 정산 초안, 야간 자동화를 같은 작업공간에서 섞으면 충돌 위험이 큽니다.
야간 자동화는 별도 worktree에서 실행하고, 결과는 diff와 AUTO_REPORT로 검토한 뒤 병합합니다.
같은 파일을 여러 worktree에서 동시에 수정하는 작업은 금지합니다.
9.4 Browser/MCP 기반 검증
Browser는 localhost 화면, 버튼, 반응형 깨짐, QR 결제 페이지 UI 확인에 사용합니다.
MCP는 Next.js 개발 서버의 build/runtime/type error, 라우트, 로그를 Codex가 직접 확인하게 하는 검증 장치입니다.
PG 운영 관리자, 카카오 운영 관리자, 실제 환불/정산 지급 화면은 자동 조작하지 않습니다.
9.5 주기적 품질 정리
매주 1회 AGENTS.md 규칙 위반, company_id/nursery_id/tablet_id 누락, mock adapter와 실제 adapter 혼동, TODO/BLOCKERS/NEXT_TASKS 누락을 점검합니다.
결제/환불/정산/보안 파일이 자동 수정되었는지 확인합니다.
불필요한 대량 리팩터링, 임시 우회, 테스트 미실행 작업을 탐지합니다.

11. Codex에 바로 넣는 프롬프트
11.1 마스터 프로젝트 이해
너는 산후조리원 폐쇄몰 기반 기업 입점형 QR 결제 쇼핑몰 시스템의 메인 개발 에이전트다. 이 프로젝트는 결제, 기업 Admin, 입점형 정산, 폐쇄몰, 산후조리원 객실/태블릿 출처 추적, QR 결제, 비회원 주문조회, 알림, 외부 재고/API 연동이 포함된 상용 커머스 시스템이다. 절대 원칙: - 추측 코딩 금지 - 운영 배포 금지 - 실결제/환불/정산 지급 금지 - Secret Key 프론트 노출 금지 - 결제/환불/정산 서버 처리 - company_id/nursery_id/tablet_id 격리 완료 시 수정 파일, 추가 파일, 테스트 결과, 실패 항목, 사람 확인 필요 항목, 다음 작업 명령어를 보고한다.
